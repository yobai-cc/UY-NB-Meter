# UY-NB-Meter 开发文档

## 1. 项目概述

`UY-NB-Meter` 当前是一个基于 Flask 的单文件接收服务，目标很简单：

- 接收 `POST /HMWSSBAPI/PostMeterReadingData`
- 按最小规则判断请求是否成功
- 将最近请求写入内存历史
- 在首页 `/` 展示请求日志

项目保持以下结构不变：

- Flask
- 单文件 `server.py`
- `render_template_string` 内联 HTML
- 内存列表 `HISTORY_DATA`

## 2. 当前架构

### 2.1 文件结构

当前核心文件：

- `server.py`：服务入口、接口逻辑、日志页模板
- `README.md`：快速使用说明
- `开发文档.md`：开发说明

### 2.2 数据存储

请求历史保存在进程内存：

- 全局变量：`HISTORY_DATA`
- 最大条数：`MAX_HISTORY = 50`

这意味着：

- 服务重启后历史清空
- 不依赖数据库
- 适合联调和简单监控

## 3. 对外接口

### 3.1 首页

- 路径：`/`
- 方法：`GET`
- 作用：展示请求日志

日志表列固定为：

- `Time`
- `Result`
- `Auth`
- `Body Length`
- `Field Count`
- `Error`
- `Raw Data`

其中 `Field Count` 只用于展示，不参与接口通过/失败判定。

### 3.2 清空历史

- 路径：`/clear`
- 方法：`GET`
- 作用：清空当前内存历史

### 3.3 上报接口

- 路径：`/HMWSSBAPI/PostMeterReadingData`
- 方法：`POST`
- Content-Type：不强制
- 请求体：任意原始字节流均可接收

## 4. 鉴权逻辑

白名单定义在 `server.py` 的 `VALID_AUTH_KEYS`：

- `Basic ZWRwOk5hdmF5dWdhMTIz`
- `Basic YWRtaW46MTIzNDU2`

当前鉴权规则为“可选鉴权”：

1. 如果没有 `Authorization` 请求头，则鉴权通过
2. 如果有 `Authorization` 请求头，且值在 `VALID_AUTH_KEYS` 中，则鉴权通过
3. 只有“请求头存在但值不在白名单中”时，才鉴权失败

鉴权失败时：

- HTTP 状态码：`401`
- 纯文本响应体：`faile`

## 5. 请求体验证逻辑

当前版本不再解析业务字段，不再校验 CAN、RSSI、电压、流量等内容。

唯一请求体校验规则：

- 使用 `request.get_data()` 获取原始字节
- 仅判断原始字节长度是否等于 `158`

即：

- `len(raw_body_bytes) == 158`：校验通过
- 其他任何长度：校验失败

包括空请求体在内，只要长度不是 `158`，都返回失败。

长度失败时：

- HTTP 状态码：`400`
- 纯文本响应体：`faile`

长度通过且鉴权通过时：

- HTTP 状态码：`200`
- 纯文本响应体：`OK`

## 6. 历史记录结构

每次请求都会写入一条历史记录，记录字段保持简单，包含但不限于：

- `Success`
- `AuthPassed`
- `LengthPassed`
- `BodyByteLength`
- `FieldCount`
- `RawData`
- `ErrorType`
- `ErrorMsg`
- `Time`
- `Pkey`

补充说明：

- `RawData` 由请求体按 UTF-8 `replace` 方式转成字符串后展示
- `FieldCount` 通过字符串按逗号分割得到，仅作调试参考
- `Auth` 页面列显示为 `PASS (missing)`、`PASS` 或 `FAIL`

## 7. 页面逻辑

首页不再区分正常视图和调试视图，也不再提供业务仪表盘。

当前页面行为：

- 固定展示请求日志表
- 默认按最新请求倒序显示
- 可通过 `/clear` 清空记录
- 页面自动刷新，便于联调查看

## 8. 运行方式

```bash
cd /home/yobai/UY-NB-Meter
python3 server.py
```

当前启动参数：

- Host：`127.0.0.1`
- Port：`15556`
- Debug：`True`

## 9. 已移除的旧逻辑

当前版本已经移除以下旧行为：

- CAN / RSSI / 流量 / 电压等业务字段解析
- 14/15 字段格式校验作为接口通过条件
- 正常视图 / 调试视图双模式
- 复杂 JSON 成功响应
- 复杂 JSON 失败响应
- 基于业务字段的错误分类和展示

## 10. 开发注意事项

- 不要引入新的框架
- 不要拆分成多文件架构
- 不要增加模板目录
- 不要接入数据库
- 如果继续扩展，优先保持 `server.py` 可读性和最小改动原则
